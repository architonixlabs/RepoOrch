- (technique) started Brain Writing Round Robin
- (idea) On install, generate first knowledge graph in every repo
- (idea) Create a master graph at the root that connects to/links the per-repo graphs
- (idea) On code change, the repo graphs (and master) get updated
- (idea) Master graph = summary/ownership knowledge: which repo is responsible for what
- (idea) Query flow: hit master graph first, it routes/propagates the query down to the relevant repo graph
- (idea) Master graph redirects to repo graphs rather than holding all detail itself
- (idea) Skills generated at both levels: per-repo skills and master-level skill(s)
- (technique) converged: affinity clustering -> MoSCoW (build-scoping)
- (decision) Two-tier skills: per-repo skill 'repo-<name>-knowledge' triggers on that repo's own domain keywords; master 'workspace-routing' skill triggers on cross-repo / which-repo questions
- (decision) Query flow: master routing skill resolves owner via registry find_owning_repos, then hands off to the owning repo's skill/graph (master redirects, does not hold repo detail)
- (decision) Knowledge build is deterministic (AST/graphify), runs in the existing session at install/setup, requires NO API key; refreshes on code change via post-commit hook + fingerprint drift
- (decision) Handoff case: dev in repo-A asks a repo-B question -> repo-A skill trigger won't match -> master skill fires -> routes to repo-B
- (decision) MoSCoW: MUST=per-repo graph + master summary + both skills at setup + no-key build + update-on-change; SHOULD=master->repo propagation, routing-poisoning bound, graph-query-first; WONT(now)=key-requiring semantic extraction
- (insight) Master graph is a routing INDEX (ownership), not a copy of repo detail - that's why it's small and why queries hit it first then propagate
- (insight) Per-repo skill trigger keywords ARE the master's routing keys (registry owns/endpoints/emits) - one source, two consumers; so routing-poisoning (security #2) is the same surface as ownership knowledge and must be bounded/advisory
